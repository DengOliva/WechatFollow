# 微信任务跟踪 MVP

这是一个在本机运行、可通过反向隧道提供公网提交页面的最小版本：

1. 在后台录入任务内容、任务期限和责任人（微信昵称）。
2. 每项任务自动生成不可猜测的专属提交链接。
3. 责任人通过链接上传资料并填写提交说明。
4. 后台查看提交状态、历史记录并下载资料。
5. 点击“手动通知”。
6. 通过 Windows UI Automation 操作已登录的微信桌面端。
7. 向责任人发送任务内容、期限和资料提交链接。

普通个人微信聊天不支持由 RPA 创建原生交互按钮。程序会发送排版后的
任务通知，并将 HTTPS 提交地址单独放在一行，微信会将其识别为可点击链接。

## 运行

先登录微信并打开微信主界面，然后在 PowerShell 中执行：

```powershell
.\run.ps1
```

浏览器访问 <http://127.0.0.1:8000>。

如需临时覆盖通知地址，启动前设置：

```powershell
$env:PUBLIC_BASE_URL="https://其他域名.example"
.\run.ps1
```

当前通知默认使用公网地址 `https://center.gxajb.site`。后台服务仍只监听
`127.0.0.1:8000`，由服务器上的 HTTPS 反向代理对外提供访问。

程序每天 08:30 提醒所有未提交资料的任务。临期或逾期任务每天
08:30 和 14:30 各提醒一次。临期范围为完成期限当天、前一天、
前两天、前三天；已提交任务不会提醒。

后台“提醒设置”可调整提醒渠道、群名称和每日时间。默认开启群消息提醒，
群名称为“中心-各项目体系负责人群”；默认关闭责任人私聊提醒。测试时可关闭
群提醒并开启私聊提醒。多个提醒时间使用英文逗号分隔。

群消息提醒会按照任务“责任人”字段执行群内 `@`。责任人姓名可以匹配群备注
中的一部分，例如责任人“张瑞东”可匹配“三澳安监部张瑞东”；同名人员应填写
更完整且可唯一匹配的群内名称。

## 注意

- 责任人应填写能在微信搜索框中准确找到的昵称或备注名。
- 通知期间不要操作鼠标键盘，微信窗口也不要被弹窗遮挡。
- 数据保存在项目目录的 `tasks.db`。
- 上传资料保存在项目目录的 `data/uploads`，单文件最大 20 MB。
- 当前版本只监听 `127.0.0.1`，不会对局域网或互联网开放。
- 新版微信部分区域是自绘控件。脚本以 UI Automation 定位和聚焦为主，在控件未暴露时使用快捷键完成输入。
